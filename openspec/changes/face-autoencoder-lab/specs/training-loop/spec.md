# 训练流程与损失跟踪规格

## ADDED Requirements

### Requirement: 损失函数定义

系统必须使用均方误差（MSE）作为重建损失函数。

#### Scenario: 计算 MSE 损失

- **当** 计算训练损失时
- **那么** 必须使用公式：LMSE = (1/N) Σᵢ ||xᵢ − x̂ᵢ||²
- **其中** xᵢ 是原始图像，x̂ᵢ 是重建图像，N 是批次大小
- **并且** 损失值必须为标量张量

#### Scenario: 损失函数初始化

- **当** 训练循环开始时
- **那么** 必须创建 nn.MSELoss() 实例
- **并且** 可以选择 reduction='mean' 计算批次平均损失

### Requirement: 优化器配置

系统必须使用 AdamW 优化器进行模型参数更新。

#### Scenario: 创建 AdamW 优化器

- **当** 初始化优化器时
- **那么** 必须使用 torch.optim.AdamW
- **并且** 默认学习率设置为 0.001
- **并且** 默认权重衰减（weight_decay）设置为 0.0001
- **并且** 支持用户自定义超参数

#### Scenario: 优化器参数

- **当** 创建优化器时
- **那么** 必须将模型的所有可训练参数传递给优化器
- **并且** 排除不需要优化的参数（如 batch norm 的 running stats）

### Requirement: 训练循环

系统必须实现完整的训练循环，支持多 epoch 训练。

#### Scenario: 单个 Epoch 训练

- **当** 执行一个训练 epoch 时
- **那么** 必须遍历训练 DataLoader 的所有批次
- **并且** 对每个批次执行：
  1. 将数据移动到设备（CPU/GPU）
  2. 前向传播获取重建图像
  3. 计算损失
  4. 反向传播计算梯度
  5. 优化器更新参数
  6. 清空梯度
- **并且** 累积每个批次的损失

#### Scenario: 训练模式设置

- **当** 开始训练时
- **那么** 必须调用 model.train() 启用训练模式
- **并且** 启用 Dropout 和 BatchNorm 的训练行为

#### Scenario: 梯度管理

- **当** 执行反向传播前
- **那么** 必须调用 optimizer.zero_grad() 清空旧梯度
- **并且** 避免梯度累积导致错误的参数更新

### Requirement: 验证循环

系统必须在每个 epoch 后执行验证，评估模型性能。

#### Scenario: 单个 Epoch 验证

- **当** 执行验证时
- **那么** 必须遍历验证 DataLoader 的所有批次
- **并且** 对每个批次执行：
  1. 将数据移动到设备
  2. 前向传播获取重建图像
  3. 计算损失
  4. 累积损失但不更新梯度
- **并且** 使用 torch.no_grad() 上下文管理器减少内存消耗

#### Scenario: 验证模式设置

- **当** 开始验证时
- **那么** 必须调用 model.eval() 启用评估模式
- **并且** 禁用 Dropout 和使用固定的 BatchNorm 统计

#### Scenario: 验证时不计算梯度

- **当** 验证模型时
- **那么** 必须使用 torch.no_grad() 禁用梯度计算
- **并且** 减少内存占用和加速计算

### Requirement: 损失记录

系统必须记录每个 epoch 的训练和验证损失。

#### Scenario: 记录训练损失

- **当** 完成一个训练 epoch 后
- **那么** 必须计算该 epoch 的平均训练损失
- **并且** 将损失值保存到列表中
- **并且** 格式为 List[float] 或 numpy array

#### Scenario: 记录验证损失

- **当** 完成一个验证 epoch 后
- **那么** 必须计算该 epoch 的平均验证损失
- **并且** 将损失值保存到列表中
- **并且** 与训练损失分开存储

#### Scenario: 实时损失显示

- **当** 完成一个 epoch 时
- **那么** 必须打印当前 epoch 编号
- **并且** 打印训练损失和验证损失
- **并且** 格式示例：`Epoch [1/20], Train Loss: 0.0234, Val Loss: 0.0256`

### Requirement: 损失曲线可视化

系统必须生成训练和验证损失随 epoch 变化的曲线图。

#### Scenario: 绘制损失曲线

- **当** 训练完成后
- **那么** 必须使用 matplotlib 绘制损失曲线
- **并且** X 轴为 epoch 编号，Y 轴为损失值
- **并且** 使用不同颜色区分训练损失和验证损失
- **并且** 添加图例说明（中文："训练损失" 和 "验证损失"）

#### Scenario: 保存损失曲线图

- **当** 生成损失曲线后
- **那么** 必须保存图像文件（默认路径：./loss_curve.png）
- **并且** 支持用户自定义保存路径和文件名
- **并且** 图像分辨率至少为 800×600 像素

#### Scenario: 损失曲线标注

- **当** 绘制损失曲线时
- **那么** 必须添加图表标题（中文："训练和验证损失曲线"）
- **并且** 添加 X 轴标签（中文："Epoch"）
- **并且** 添加 Y 轴标签（中文："损失值"）
- **并且** 显示网格线以便读取数值

### Requirement: 训练超参数

系统必须支持配置训练超参数。

#### Scenario: 设置 Epoch 数量

- **当** 初始化训练时
- **那么** 必须支持自定义 epoch 数量（默认 20-30）
- **并且** 允许用户指定任意正整数

#### Scenario: 设置批次大小

- **当** 创建 DataLoader 时
- **那么** 必须支持自定义批次大小（默认 64）
- **并且** 确保批次大小不超过可用内存

#### Scenario: 设置学习率

- **当** 创建优化器时
- **那么** 必须支持自定义学习率（默认 0.001）
- **并且** 允许用户指定任意正浮点数

### Requirement: 模型检查点

系统必须支持保存训练过程中的最佳模型。

#### Scenario: 保存最佳模型

- **当** 验证损失降低时
- **那么** 必须保存当前模型参数到文件
- **并且** 使用默认路径：./best_model.pth
- **并且** 支持用户自定义保存路径

#### Scenario: 保存最终模型

- **当** 训练完成后
- **那么** 必须保存最后的模型参数到文件
- **并且** 使用默认路径：./final_model.pth
- **并且** 包含模型状态字典和优化器状态字典

#### Scenario: 加载保存的模型

- **当** 需要恢复训练或评估时
- **那么** 必须支持从检查点文件加载模型参数
- **并且** 恢复模型权重和优化器状态

### Requirement: 训练时间监控

系统必须记录和显示训练时间。

#### Scenario: 记录每个 Epoch 时间

- **当** 完成一个 epoch 时
- **那么** 必须记录该 epoch 的耗时
- **并且** 打印时间信息（中文："耗时: X 秒"）

#### Scenario: 记录总训练时间

- **当** 完成所有训练时
- **那么** 必须计算并打印总训练时间
- **并且** 格式为分钟和秒（如："总训练时间: 15 分 30 秒"）

### Requirement: 设备管理

系统必须自动检测和使用可用的计算设备。

#### Scenario: 检测 GPU 可用性

- **当** 初始化训练时
- **那么** 必须检测 CUDA 是否可用
- **并且** 优先使用 GPU，否则使用 CPU
- **并且** 打印当前使用的设备信息

#### Scenario: 移动数据到设备

- **当** 训练或验证时
- **那么** 必须将输入数据和模型移动到同一设备
- **并且** 确保所有张量在同一设备上计算
