# 自编码器模型架构规格

## ADDED Requirements

### Requirement: 编码器架构

系统必须实现具有 4 层卷积网络的编码器，将图像压缩为潜在向量。

#### Scenario: 编码器卷积层结构

- **当** 编码器接收输入图像时
- **那么** 必须使用以下卷积层配置：
  - 第 1 层：输入通道 3，输出通道 32
  - 第 2 层：输入通道 32，输出通道 64
  - 第 3 层：输入通道 64，输出通道 128
  - 第 4 层：输入通道 128，输出通道 256
- **并且** 每层的卷积核大小必须为 3×3
- **并且** 步幅（stride）必须为 2
- **并且** 填充（padding）必须为 1

#### Scenario: 编码器空间维度变化

- **当** 图像通过编码器各层时
- **那么** 空间分辨率必须按以下顺序递减：
  - 输入：64×64
  - 第 1 层后：32×32
  - 第 2 层后：16×16
  - 第 3 层后：8×8
  - 第 4 层后：4×4
- **并且** 最终特征图尺寸必须为 (256, 4, 4)

#### Scenario: 编码器激活函数

- **当** 数据通过每个卷积层后
- **那么** 必须立即应用 ReLU 激活函数
- **并且** 使用 inplace=True 优化内存使用（可选）

### Requirement: 潜在向量压缩

编码器必须将特征图展平并通过全连接层压缩为潜在向量。

#### Scenario: 特征图展平

- **当** 完成最后一个卷积层时
- **那么** 必须将 (256, 4, 4) 的特征图展平为 4096 维向量
- **并且** 展平操作必须保持所有通道信息

#### Scenario: 生成潜在向量

- **当** 展平的向量通过全连接层时
- **那么** 必须将其映射到可配置的潜在维度（默认 100）
- **并且** 输出必须为 100 维的潜在向量
- **并且** 必须支持在初始化时自定义潜在维度（32、100、256 等）

### Requirement: 解码器架构

系统必须实现解码器，将潜在向量重建为原始图像。

#### Scenario: 解码器全连接层扩展

- **当** 解码器接收潜在向量时
- **那么** 必须通过全连接层将潜在维度扩展到 256×4×4 = 4096
- **并且** 将结果重塑为 (256, 4, 4) 的特征图

#### Scenario: 解码器上采样层结构

- **当** 特征图通过解码器时
- **那么** 必须使用 4 层转置卷积进行上采样
- **并且** 空间分辨率必须按以下顺序递增：
  - 输入：4×4
  - 第 1 层后：8×8
  - 第 2 层后：16×16
  - 第 3 层后：32×32
  - 第 4 层后：64×64
- **并且** 通道数必须按以下顺序递减：
  - 输入：256
  - 第 1 层后：128
  - 第 2 层后：64
  - 第 3 层后：32
  - 第 4 层后：3（RGB 通道）

#### Scenario: 解码器上采样方法

- **当** 执行上采样操作时
- **那么** 必须使用双线性插值（Bilinear Upsampling）
- **或者** 使用转置卷积（ConvTranspose2d）配合合适的参数
- **并且** 保证输出图像尺寸精确为 64×64

### Requirement: 解码器激活函数

解码器必须使用适当的激活函数确保输出范围正确。

#### Scenario: 中间层激活函数

- **当** 数据通过解码器的上采样层时
- **那么** 除了最后一层外，每层后必须应用 ReLU 激活函数
- **并且** 激活函数必须保持梯度流动顺畅

#### Scenario: 输出层激活函数

- **当** 数据通过最后一层时
- **那么** 必须应用 Sigmoid 激活函数
- **并且** 确保输出值范围在 [0, 1] 之间
- **并且** 与输入图像的归一化范围匹配

### Requirement: 模型接口

系统必须提供清晰的模型接口用于前向传播。

#### Scenario: 前向传播

- **当** 输入一批图像到模型时
- **那么** 模型必须返回重建的图像
- **并且** 输出形状必须与输入形状完全相同 (batch_size, 3, 64, 64)
- **并且** 支持同时返回潜在向量（用于分析）

#### Scenario: 编码器单独使用

- **当** 仅需要编码器提取特征时
- **那么** 必须提供独立访问编码器的方法
- **并且** 返回指定维度的潜在向量

#### Scenario: 解码器单独使用

- **当** 给定潜在向量需要生成图像时
- **那么** 必须提供独立访问解码器的方法
- **并且** 从潜在向量生成重建图像

### Requirement: 模型参数配置

系统必须支持通过参数配置模型结构。

#### Scenario: 可配置潜在维度

- **当** 初始化模型时
- **那么** 必须允许用户指定潜在维度（latent_dim）
- **并且** 默认值为 100
- **并且** 支持任意正整数（如 32、64、128、256）

#### Scenario: 模型初始化

- **当** 创建模型实例时
- **那么** 必须自动初始化所有层的权重
- **并且** 使用合适的初始化方法（如 Xavier 或 Kaiming 初始化）
