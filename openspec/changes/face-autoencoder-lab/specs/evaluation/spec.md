# 模型评估与实验分析规格

## ADDED Requirements

### Requirement: 重建质量可视化

系统必须能够对比显示原始图像和重建图像。

#### Scenario: 可视化单个批次对比

- **当** 评估模型时
- **那么** 必须从验证集中选取一个批次（如 8-16 张图像）
- **并且** 在图中并排显示原始图像和重建图像
- **并且** 使用奇数行显示原始图像，偶数行显示重建图像
- **或者** 使用左右两列对比显示

#### Scenario: 保存对比图像

- **当** 生成对比图后
- **那么** 必须保存为高分辨率图像文件
- **并且** 默认路径为：./reconstruction_comparison.png
- **并且** 分辨率至少为 1024×1024 像素
- **并且** 图像标题使用中文（如 "重建质量对比"）

#### Scenario: 计算重建指标

- **当** 生成对比图时
- **那么** 必须计算并显示该批次的平均 MSE 损失
- **并且** 可选：计算 PSNR（峰值信噪比）和 SSIM（结构相似性）

### Requirement: 不同潜在维度对比实验

系统必须支持训练和比较不同潜在维度的模型。

#### Scenario: 训练多个模型

- **当** 执行对比实验时
- **那么** 必须支持训练至少三个不同潜在维度的模型：32、100、256
- **并且** 使用相同的训练超参数和数据集
- **并且** 为每个模型创建独立的保存目录

#### Scenario: 保存实验结果

- **当** 完成每个模型的训练时
- **那么** 必须保存模型权重到对应目录（如：./models/latent_dim_32/）
- **并且** 保存各自的损失曲线图
- **并且** 记录最终的训练和验证损失

#### Scenario: 对比重建质量

- **当** 比较不同潜在维度时
- **那么** 必须为每个模型生成重建对比图
- **并且** 使用相同的输入图像进行公平比较
- **或者** 在一张大图中并排显示三个模型的重建结果

#### Scenario: 损失曲线对比

- **当** 所有模型训练完成后
- **那么** 必须在一张图中绘制所有模型的损失曲线
- **并且** 使用不同颜色区分不同潜在维度
- **并且** 添加图例说明（中文："潜在维度=32"、"潜在维度=100"、"潜在维度=256"）
- **并且** 保存为对比图（./latent_dim_comparison.png）

#### Scenario: 生成对比表格

- **当** 完成对比实验时
- **那么** 必须生成包含以下信息的表格：
  - 潜在维度
  - 最终训练损失
  - 最终验证损失
  - 训练时间
  - 模型参数数量
- **并且** 以 Markdown 或 CSV 格式保存

### Requirement: 潜在空间分析

系统必须提供分析潜在空间存储信息的工具。

#### Scenario: 可视化潜在向量

- **当** 分析潜在空间时
- **那么** 必须提取一批图像的潜在向量表示
- **并且** 使用 t-SNE 或 PCA 将高维潜在向量降维到 2D
- **并且** 绘制散点图显示样本在潜在空间的分布

#### Scenario: 潜在向量统计

- **当** 分析潜在向量时
- **那么** 必须计算并显示统计信息：
  - 每个维度的均值和标准差
  - 潜在向量的 L2 范数分布
  - 维度之间的相关性矩阵（可选）

#### Scenario: 潜在空间插值

- **当** 探索潜在空间结构时
- **那么** 必须支持在两个样本的潜在向量之间进行线性插值
- **并且** 对插值点进行解码生成图像
- **并且** 可视化插值序列（如 10 个点）显示潜在空间的连续性

### Requirement: 随机生成分析

系统必须探索从随机潜在向量生成图像的效果。

#### Scenario: 从随机向量生成图像

- **当** 评估生成能力时
- **那么** 必须从标准正态分布采样随机潜在向量
- **并且** 通过解码器生成图像
- **并且** 可视化生成的样本（8×8 网格显示 64 张图像）

#### Scenario: 分析生成质量

- **当** 生成随机图像时
- **那么** 必须记录并分析：
  - 生成图像的清晰度
  - 人脸特征的完整性（眼睛、鼻子、嘴巴等）
  - 图像的多样性和一致性
- **并且** 在实验报告中描述观察结果

#### Scenario: 潜在向量范围探索

- **当** 探索生成效果时
- **那么** 必须尝试不同范围的潜在向量值：
  - 标准正态分布（均值 0，标准差 1）
  - 训练数据的潜在向量分布
  - 不同范围（如 [-3, 3]、[-5, 5]）
- **并且** 对比生成质量

### Requirement: 实验报告生成

系统必须生成包含分析和结论的实验报告。

#### Scenario: 回答分析问题

- **当** 生成实验报告时
- **那么** 必须回答以下问题并提供证据：
  1. **潜在空间存储的信息类型**：
     - 哪些人脸特征被编码（性别、表情、角度等）
     - 潜在维度如何影响编码信息量
     - 提供可视化和定量分析

  2. **生成新样本的局限性**：
     - 为什么自编码器能重建但不能可靠生成新样本
     - 随机潜在向量生成图像的质量问题
     - 与 VAE 等生成模型的对比分析

  3. **潜在维度对重建质量的影响**：
     - 32、100、256 维度的重建质量对比
     - 损失曲线和定量指标
     - 计算效率和存储空间的权衡

#### Scenario: 保存 Markdown 报告

- **当** 生成实验报告时
- **那么** 必须保存为 Markdown 格式（./experiment_report.md）
- **并且** 包含以下章节：
  - 实验设置（数据集、模型架构、超参数）
  - 训练过程（损失曲线、训练时间）
  - 重建质量评估（可视化、定量指标）
  - 潜在空间分析（可视化、统计）
  - 不同潜在维度对比（表格、图表）
  - 结论和讨论（回答三个分析问题）
- **并且** 插入所有生成的图像链接

#### Scenario: 支持中英文报告

- **当** 生成报告时
- **那么** 必须支持中文或英文输出
- **并且** 默认使用中文
- **并且** 允许用户通过参数切换语言

### Requirement: 评估工具接口

系统必须提供统一的评估接口。

#### Scenario: 加载训练好的模型

- **当** 开始评估时
- **那么** 必须提供函数加载指定路径的模型权重
- **并且** 支持加载不同潜在维度的模型
- **并且** 验证模型架构与权重文件匹配

#### Scenario: 批量评估

- **当** 评估多个模型时
- **那么** 必须支持批量评估接口
- **并且** 自动遍历指定目录下的所有模型
- **并且** 生成统一的对比报告

#### Scenario: 交互式评估

- **当** 用户需要交互式探索时
- **那么** 必须提供 Jupyter Notebook 友好的接口
- **并且** 支持逐步执行和可视化中间结果
- **并且** 允许用户自定义评估参数

### Requirement: 实验可复现性

系统必须确保实验结果可复现。

#### Scenario: 设置随机种子

- **当** 开始训练或评估时
- **那么** 必须设置所有随机种子（torch、numpy、random）
- **并且** 使用固定的种子值（如 42）
- **并且** 在日志中记录使用的种子

#### Scenario: 记录实验配置

- **当** 执行实验时
- **那么** 必须保存完整的配置信息：
  - 数据集路径和划分
  - 模型架构参数
  - 训练超参数
  - 随机种子
  - 使用的设备
- **并且** 保存为 JSON 或 YAML 文件（./experiment_config.json）

#### Scenario: 记录环境信息

- **当** 执行实验时
- **那么** 必须记录环境信息：
  - Python 版本
  - PyTorch 版本
  - CUDA 版本（如果使用 GPU）
  - 操作系统信息
- **并且** 保存到实验配置文件中
