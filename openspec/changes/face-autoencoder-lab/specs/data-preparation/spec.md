# 数据准备与可视化规格

## ADDED Requirements

### Requirement: 加载 CelebA 数据集

系统必须能够从本地 `./img_align_celeba/` 目录加载 CelebA Align&Cropped 图像数据集。

#### Scenario: 成功加载图像文件

- **当** 系统初始化数据加载器时
- **那么** 系统必须从指定目录读取所有 JPEG 格式的人脸图像文件
- **并且** 必须记录成功加载的图像数量

#### Scenario: 处理不存在的数据目录

- **当** 指定的数据目录不存在时
- **那么** 系统必须抛出明确的错误信息
- **并且** 提示用户检查数据集路径

### Requirement: 图像预处理

系统必须对所有图像执行统一的预处理操作，包括调整大小和归一化。

#### Scenario: 调整图像尺寸

- **当** 加载原始图像时
- **那么** 系统必须将每张图像调整为 64×64 像素
- **并且** 使用双线性插值保持图像质量

#### Scenario: 归一化像素值

- **当** 图像被加载为张量后
- **那么** 系统必须将像素值从 [0, 255] 转换到 [0, 1] 范围
- **并且** 转换为 float32 数据类型

### Requirement: 训练集和验证集划分

系统必须将数据集划分为训练集和验证集，用于模型训练和验证。

#### Scenario: 创建训练 DataLoader

- **当** 系统创建训练数据加载器时
- **那么** 必须使用 80% 的数据作为训练集
- **并且** 支持批量大小配置（默认 batch_size=64）
- **并且** 必须在每个 epoch 内随机打乱数据顺序

#### Scenario: 创建验证 DataLoader

- **当** 系统创建验证数据加载器时
- **那么** 必须使用 20% 的数据作为验证集
- **并且** 支持批量大小配置（默认 batch_size=64）
- **并且** 验证集不需要打乱顺序

### Requirement: 批次图像可视化

系统必须提供批次图像的可视化功能，用于数据检查和结果展示。

#### Scenario: 可视化一个批次

- **当** 调用可视化函数时
- **那么** 系统必须生成 8×8 网格显示 64 张图像
- **并且** 每个子图像必须显示标题（可选）
- **并且** 图像标题必须使用中文（如 "训练数据样本"）
- **并且** 必须保存或显示生成的图像

#### Scenario: 保存可视化结果

- **当** 用户指定保存路径时
- **那么** 系统必须将可视化图像保存为 PNG 或 JPG 格式
- **并且** 分辨率至少为 800×800 像素以保证清晰度

### Requirement: 数据转换管道

系统必须使用 torchvision 的转换管道来高效处理图像。

#### Scenario: 应用转换管道

- **当** 创建数据集时
- **那么** 必须使用 `transforms.Compose` 组合所有转换操作
- **并且** 转换顺序必须为：Resize → ToTensor → 归一化（可选）
- **并且** 所有转换必须在数据加载时自动应用

#### Scenario: 支持自定义转换

- **当** 用户需要额外的数据增强时
- **那么** 系统必须支持添加可选的转换（如随机翻转、旋转）
- **并且** 这些增强仅应用于训练集，不应用于验证集
